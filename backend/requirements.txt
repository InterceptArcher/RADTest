# HTTP client. Must be >=0.26 (the version that ADDED the `proxy` kwarg to
# httpx.Client): supabase's transitive deps (gotrue/postgrest/etc.) pass `proxy=`
# into httpx.Client, so with httpx <0.26 every create_client() call threw
# "Client.__init__() got an unexpected keyword argument 'proxy'" and the backend
# silently failed ALL Supabase writes (token persistence, job results, api costs).
# Kept <0.28 so the deprecated `proxies` kwarg is still accepted; satisfies
# anthropic/openai/gql too. PAIRED WITH supabase>=2.9 below: supabase 2.3.4 pinned
# httpx<0.26 (an impossible build), so the supabase bump is what unblocks 0.27.2.
